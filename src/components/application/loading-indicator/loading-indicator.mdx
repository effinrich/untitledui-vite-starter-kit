# LoadingIndicator

The `LoadingIndicator` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={LoadingIndicatorStories.Default} />

```tsx
<LoadingIndicator size="md">
  Content
</LoadingIndicator>
```

### Sizes

Available in 4 sizes:

<Canvas of={LoadingIndicatorStories.Sizes} />

```tsx
<LoadingIndicator size="sm">sm</LoadingIndicator>
<LoadingIndicator size="md">md</LoadingIndicator>
<LoadingIndicator size="lg">lg</LoadingIndicator>
<LoadingIndicator size="xl">xl</LoadingIndicator>
```

## Props

<ArgTypes of={LoadingIndicatorStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `"line-simple" \| "line-spinner" \| "dot-circle"` | `-` | - |
| `size` | `"sm" \| "md" \| "lg" \| "xl"` | `-` | - |
| `label` | `string` | `-` | - |


## Accessibility

The `LoadingIndicator` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx
<LoadingIndicator
  size="sm"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive LoadingIndicator
</LoadingIndicator>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use md size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
