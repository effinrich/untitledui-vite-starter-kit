# PaginationLine

The `PaginationLine` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={PaginationLineStories.Default} />

```tsx
<PaginationLine size="lg">
  Content
</PaginationLine>
```

### Sizes

Available in 2 sizes:

<Canvas of={PaginationLineStories.Sizes} />

```tsx
<PaginationLine size="md">md</PaginationLine>
<PaginationLine size="lg">lg</PaginationLine>
```

## Props

<ArgTypes of={PaginationLineStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"md" \| "lg"` | `-` | - |
| `framed` | `boolean` | `-` | - |


## Accessibility

The `PaginationLine` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx
<PaginationLine
  size="md"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive PaginationLine
</PaginationLine>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use lg size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
