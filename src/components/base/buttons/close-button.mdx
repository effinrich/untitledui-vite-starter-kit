# CloseButton

The `CloseButton` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={CloseButtonStories.Default} />

```tsx
<CloseButton size="sm">
  Content
</CloseButton>
```

### Sizes

Available in 4 sizes:

<Canvas of={CloseButtonStories.Sizes} />

```tsx
<CloseButton size="xs">xs</CloseButton>
<CloseButton size="sm">sm</CloseButton>
<CloseButton size="md">md</CloseButton>
<CloseButton size="lg">lg</CloseButton>
```

## Props

<ArgTypes of={CloseButtonStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `theme` | `"light" \| "dark"` | `-` | - |
| `size` | `"xs" \| "sm" \| "md" \| "lg"` | `-` | - |
| `label` | `string` | `-` | - |


## Accessibility

The `CloseButton` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx
<CloseButton
  size="xs"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive CloseButton
</CloseButton>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use sm size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
