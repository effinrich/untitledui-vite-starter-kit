# PinInput

The `PinInput` component provides configurable sizes, disabled state support.

## Usage

### Basic Example

<Canvas of={PinInputStories.Default} />

```tsx
<PinInput>
  Content
</PinInput>
```

## Props

<ArgTypes of={PinInputStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"sm",` | `-` | - |
| `id` | `"",` | `-` | - |
| `disabled` | `false,` | `-` | - |


## Accessibility

The `PinInput` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers
- Correctly announces disabled state

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
