# MultiSelect

The `MultiSelect` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={MultiSelectStories.Default} />

```tsx
<MultiSelect size="md">
  Content
</MultiSelect>
```

### Sizes

Available in 2 sizes:

<Canvas of={MultiSelectStories.Sizes} />

```tsx
<MultiSelect size="sm">sm</MultiSelect>
<MultiSelect size="md">md</MultiSelect>
```

## Accessibility

The `MultiSelect` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx
<MultiSelect
  size="sm"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive MultiSelect
</MultiSelect>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use md size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
