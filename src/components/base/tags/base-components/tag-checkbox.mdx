# TagCheckbox

The `TagCheckbox` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={TagCheckboxStories.Default} />

```tsx
<TagCheckbox size="md">
  Content
</TagCheckbox>
```

### Sizes

Available in 3 sizes:

<Canvas of={TagCheckboxStories.Sizes} />

```tsx
<TagCheckbox size="sm">sm</TagCheckbox>
<TagCheckbox size="md">md</TagCheckbox>
<TagCheckbox size="lg">lg</TagCheckbox>
```

## Props

<ArgTypes of={TagCheckboxStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"sm" \| "md" \| "lg"` | `-` | - |
| `className` | `string` | `-` | - |
| `isFocused` | `boolean` | `-` | - |
| `isSelected` | `boolean` | `-` | - |
| `isDisabled` | `boolean` | `-` | - |


## Accessibility

The `TagCheckbox` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx
<TagCheckbox
  size="sm"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive TagCheckbox
</TagCheckbox>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use md size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
