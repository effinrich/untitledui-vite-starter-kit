# TagCloseX

The `TagCloseX` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={TagCloseXStories.Default} />

```tsx
<TagCloseX size="md">
  Content
</TagCloseX>
```

### Sizes

Available in 3 sizes:

<Canvas of={TagCloseXStories.Sizes} />

```tsx
<TagCloseX size="sm">sm</TagCloseX>
<TagCloseX size="md">md</TagCloseX>
<TagCloseX size="lg">lg</TagCloseX>
```

## Props

<ArgTypes of={TagCloseXStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `size` | `"sm" \| "md" \| "lg"` | `-` | - |
| `className` | `string` | `-` | - |


## Accessibility

The `TagCloseX` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Responsive Design

Use CSS media queries for responsive behavior:

```tsx

  size="sm"
  style={{ width: '100%' }}
  className="responsive-component"
>
  Responsive TagCloseX
</TagCloseX>

<style>
  @media (min-width: 768px) {
    .responsive-component {
      width: auto;
      /* Use md size */
    }
  }
</style>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
