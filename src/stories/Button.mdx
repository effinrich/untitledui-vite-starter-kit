# Button

The `Button` component provides configurable sizes.

## Usage

### Basic Example

<Canvas of={ButtonStories.Default} />

```tsx
<Button size="medium">
  Content
</Button>
```

### Sizes

Available in 3 sizes:

<Canvas of={ButtonStories.Sizes} />

```tsx
<Button size="small">small</Button>
<Button size="medium">medium</Button>
<Button size="large">large</Button>
```

## Props

<ArgTypes of={ButtonStories} />

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `primary` | `boolean` | `-` | - |
| `backgroundColor` | `string` | `-` | - |
| `size` | `'small' \| 'medium' \| 'large'` | `-` | - |
| `label` | `string` | `-` | - |
| `onClick` | `() => void` | `-` | - |


## Accessibility

The `Button` component follows WAI-ARIA guidelines:

- Supports keyboard navigation
- Includes proper ARIA attributes
- Compatible with screen readers

### Keyboard Interactions

| Key | Description |
|-----|-------------|
| `Tab` | Moves focus to the component |
| `Enter` | Activates the component |
| `Space` | Activates the component |

## Composition

### With Other Components

```tsx
<div className="flex flex-col gap-4">
  <Button>
    First item
  </Button>
  <Button>
    Second item
  </Button>
</div>
```

## Responsive Design

Use Tailwind responsive prefixes:

```tsx
<Button
  className="small md:medium w-full md:w-auto"
>
  Responsive Button
</Button>
```

## Best Practices

1. **Accessibility**: Always provide meaningful content or aria-label
3. **Sizing**: Use consistent sizes within the same section of your UI
4. **Composition**: Combine with other components using Stack or Flex for layouts
5. **Performance**: Avoid creating new callback functions on each render
